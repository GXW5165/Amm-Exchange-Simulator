## 主题：去中心化金融（DeFi）核心逻辑仿真系统
**课程名称：** 金融软件工程实验  
**适用学期：** 2025-2026 学年 第二学期  
**发布日期：** 2026 年 3 月 6 日（第一周）  
**截止日期：** 2026 年 6 月 19 日

---

### 1. 项目背景与目标
#### 1.1 背景
去中心化金融（DeFi）的本质是**用代码执行的金融协议**。其核心价值不在于“是否上链”，而在于**透明、可编程、无需许可的金融逻辑**。本课程采用“离线仿真”方式，聚焦 DeFi 的算法与经济模型本质。

#### 1.2 教学目标
通过本项目，学生将掌握：

1. **金融建模能力：** 将 AMM、借贷、清算等 DeFi 机制转化为可计算的数学模型与代码逻辑。
2. **仿真系统设计：** 构建支持多用户、多资产、时间序列事件的离散事件仿真框架。
3. **工程化实践：** 遵循软件工程规范，包括模块化设计、单元测试、性能分析、文档编写。
4. **风险量化意识：** 通过压力测试、极端行情模拟，理解 DeFi 协议的脆弱性与防御策略。

---

### 2. 项目主题与范围
#### 2.1 选题原则
学生需**自行组队(3-4人)并确定具体仿真主题**。选题应围绕 DeFi 核心机制，用代码模拟其行为。

+ **核心要求：** 必须实现**可交互的仿真系统**（命令行或 Web 界面），支持参数配置、事件触发、结果可视化。
+ **聚焦 MVP(****<font style="color:rgb(29, 29, 31);">Minimum Viable Product</font>****)：** 不追求完整生态，聚焦 1-2 个核心机制的深度模拟（如：仅模拟 AMM 的价格冲击，或仅模拟借贷的清算逻辑）。

#### 2.2 推荐方向
| 方向 | 仿真重点 | 技术挑战 |
| :--- | :--- | :--- |
| **AMM 交易所仿真** | 恒定乘积公式、滑点计算、LP 收益、无常损失 | 数值精度、大额交易冲击模拟 |
| **借贷协议仿真** | 抵押率、清算阈值、利率模型、坏账处理 | 多资产联动、极端行情压力测试 |
| **收益策略回测** | 自动复投、多协议套利、Gas 成本模拟（可抽象） | 历史数据接入、策略绩效评估 |
| **预言机攻击模拟** | 价格操纵、时间加权平均价（TWAP）防御 | 攻击路径建模、防御机制对比 |
| **DAO 治理仿真** | 投票权重、提案通过条件、治理攻击（如闪电贷投票） | 博弈逻辑、时间延迟模拟 |


#### 2.3 禁止事项
+ **禁止“伪仿真”：** 不能仅用 Excel 公式或静态计算，必须有**可执行、可交互、可复现**的代码系统。
+ **禁止直接调用 DeFi 主网 API 作为核心逻辑：** 允许引用历史链上数据作为输入，但核心协议逻辑必须本地实现。
+ **禁止抄袭：** 参考开源逻辑需明确标注，核心算法需有自主实现或改进。

### 3. 项目交付物
#### 3.1 源代码仓库 (GitHub)
+ 清晰的项目结构，如：

```plain
project/
├── src/           # 核心逻辑（AMM/Lending等）
├── simulator/     # 仿真引擎（事件调度、用户代理）
├── tests/         # 单元测试与集成测试
├── data/          # 示例输入数据（如历史价格）
├── frontend/      # 交互界面（如有）
├── README.md      # 快速启动指南
└── requirements.txt / CMakeLists.txt
```

#### 3.2 技术报告 (PDF, ≥3000 字)
1. **问题定义：** 模拟的 DeFi 机制是什么？解决什么金融问题？
2. **数学模型：** 核心公式推导（如 AMM 的 `x*y=k`、借贷的抵押率计算）。
3. **系统架构：** 模块划分、事件流图、类/数据结构设计。
4. **仿真设计：** 时间推进方式、用户行为建模、随机性控制（种子固定）。
5. **实验与验证：** 
    - 基础功能测试（如：交换后池子余额是否守恒）
    - 极端场景测试（如：价格暴跌时清算是否触发）
    - 对比分析（如：不同利率模型下的协议收益）
6. **团队分工与反思：** 每人贡献 + 项目难点与改进方向。

#### 3.3 演示材料
+ **可执行Demo：** 提供一键运行脚本（如 `python main.py --config demo.yaml`）
+ **演示视频 (3-5 分钟)：** 录屏展示系统运行、参数调整、结果输出
+ **答辩 PPT (10 分钟)：** 重点讲解模型设计、关键代码、实验结论

#### 示例 2：借贷协议清算模拟器（C++ 思路）
```cpp
struct LoanPosition {
    double collateral;   // 抵押品数量
    double debt;         // 借出稳定币数量
    double collateral_price; // 实时价格（可来自模拟预言机）
    
    bool is_liquidatable(double liquidation_threshold) const {
        return (collateral * collateral_price) / debt < liquidation_threshold;
    }
};

class LiquidationEngine {
public:
    // 模拟价格暴跌时，批量检查并执行清算
    void run_liquidation_cycle(vector<LoanPosition>& positions, double new_price);
};
```

**扩展方向：** 模拟清算奖励、坏账分摊、不同清算策略（荷兰拍卖 vs 固定折扣）的效果对比。
